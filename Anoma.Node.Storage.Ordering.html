        Anoma.Node.Storage.Ordering (Anoma v0.11.0)

        I am a simple mnesia-backed key-value store in an anoma node.

        Currently we do not have a way of propagating what keys users want to store, thus we take the following approach to deterministic state reading:

        1. We keep a next_order, which represents the next_order a transaction will have. From here the scry reads specifically [next_order, :key_space | 0] to get a map of the current keys saved for this node.

        2. We keep a hash_to_order to cache the id => order mapping
